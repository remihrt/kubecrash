# GitOps — Flux CD

Flux CD watches this repository and reconciles the cluster state to match Git. No manual `kubectl apply` — push a commit, Flux takes care of the rest.

## Structure

```
03-gitops/
├── clusters/kubecrash/        # Flux wiring for this cluster
│   ├── flux-system/           # Auto-generated by flux bootstrap — do not edit
│   ├── apps.yaml              # Entry point: points Flux at apps/
│   └── infrastructure.yaml   # Entry point: points Flux at infrastructure/
├── apps/
│   ├── kustomization.yaml     # Root: lists all workloads
│   └── <workload>/            # One directory per workload (raw manifests)
└── infrastructure/
    ├── kustomization.yaml     # Root: lists all infrastructure components
    └── <component>/           # One directory per component (Helm-based)
```

## Layers

**`apps/`** — stateless workloads deployed as raw Kubernetes manifests (Deployments, Services).

**`infrastructure/`** — cluster-level services deployed as Helm releases (monitoring, ingress, cert-manager, etc.).

## Adding a workload (raw manifests)

1. Create `apps/<name>/` with `namespace.yaml`, `deployment.yaml`, `service.yaml`, and `kustomization.yaml`
2. Add `- <name>` to `apps/kustomization.yaml`
3. `git commit && git push`
4. Flux applies it within 1 minute

Force an immediate sync: `flux reconcile kustomization apps --with-source`

## Adding infrastructure (Helm-based)

1. Create `infrastructure/<name>/` with `namespace.yaml`, `helmrepository.yaml`, `helmrelease.yaml`, and `kustomization.yaml`
2. Add `- <name>` to `infrastructure/kustomization.yaml`
3. `git commit && git push`
4. Flux applies it within 1 minute (chart install may take several minutes)

Force an immediate sync: `flux reconcile kustomization infrastructure --with-source`

## Bootstrap (one-time)

```bash
export GITHUB_TOKEN=ghp_<token_with_repo_scope>
flux bootstrap github \
  --owner=remihrt \
  --repository=kubecrash \
  --branch=main \
  --path=03-gitops/clusters/kubecrash \
  --personal
```

## Multi-arch requirement

All images must support `linux/amd64` and `linux/arm64`. Verify before adding:

```bash
docker buildx imagetools inspect <image>:<tag>
```
